Problem Statement:

Develop a package delivery status tracker.

A courier company requires a simple tracking system that allows user to store and manage package delivery information.
The system should allow adding packages,updating delivery status,viewing package details,listing all packages,and deleting packages.

The goal is to design a modular Python project using CSV for storage and a command-line interface for interaction.


Scope of the project:

The system includes:

Package creation

Delivery status updates

Viewing package details

Viewing all packages

Deleting packages


Target users:

Students learning file handling & modular programming

Small businesses needing simple offline package tracking

Beginners practicing Python CLI applications


High-Level Features:

Add packages with tracking ID, sender, receiver, notes

Update delivery status (Booked → In Transit → Out for Delivery → Delivered)

View a single package’s details

View all packages in table-like format

Delete packages

Auto-logging system for status history

Persistent CSV storage
